# System Design: Combinatorial Mixture-of-Experts (MoE) Orchestration

## 1. Mathematical Objective
The PCA model utilizes a "Cognitive Router" to distribute latent hidden states (H_t) among a sparse selection of expert sub-networks (E_n). The primary goal is to maximize computational throughput by only activating 2 out of 16 experts per token, achieving O(1/N) FLOP efficiency for the active forward pass while maintaining O(N) parameter capacity for knowledge retention.

## 2. The Routing Mechanics
The router R(x) is defined by a learnable weight matrix W_r.
- **Top-K Selection:** The input vector is multiplied by W_r; the top 2 indices (k=2) are selected.
- **Expert Dispatch:** A gating probability G(x) is calculated via a Softmax over the indices.
- **Normalization:** To prevent "expert collapse" (where one expert absorbs all gradients), a Load Balancing Loss (L_b) is added to the total training loss.

## 3. Data Flow and Latent Tunnels
1. **Input H_t:** The token embedding from the attention layer enters the router.
2. **Scoring:** R(x) assigns a "specialization score" to each expert based on the input's semantic structure (not factual content).
3. **Wait/Forward:** The input is dispatched to E_1 and E_3.
4. **Weighted Sum:** The outputs are combined: Y = G_1(x)E_1(x) + G_3(x)E_3(x).

## 4. Constraints and Failure Modes
- **Constraint 1 (Memory):** Total parameters = 3B. Active parameters per token = 450M.
- **Failure Mode A (Expert Collapse):** If L_b is too low, the router defaults to a single "generalist" expert. This destroys the MoE benefit.
- **Failure Mode B (Dispatch Jitter):** High variance in routing decisions between adjacent tokens in a sequence creates noise in the gradient descent path.

## 5. Implementation Requirements
Any script implementing this router must:
- Implement a `NoisyTopK` to encourage exploration during Phase 1 pre-training.
- Ensure CUDA kernels for dispatch are vectorized to minimize latencies between the CPU orchestrator and GPU memory bank.
